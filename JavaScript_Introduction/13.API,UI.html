UI란? User Interface의 약자


시스템과 사용자의 접점에 존재하면서 사용자가 생각하는것을 컴퓨터에게 전달하는 입력장치들
혹은 컴퓨터가 계산한 값을이나 상태를 사용자에게 전달하는 출력장치들에 해당되는 접점, 중계자에 해당되는 것


API란? Application Programming Interface의 약자


코드의 형태를 띄고있는 인터페이스
우리는 웹브라우저가 제공하고있는 조작방법을 다양한 방법으로 결합해서 애플리케이션을 만드는것이다
alert과 같은 인터페이스를 사용자가 사용하는 인터페이스와 구분하기 위해서 API라고 부른다
즉,

프로그램이 동작하는 환경을 제어하기 위해 환경에서 제공되는 조작장치


기반이 되는 시스템이 우리에게 제공한 인터페이스를 응용해서 프로그램을 만드는 것
웹개발자들은 API를 통해서 웹브라우저를 제어한다
개발자와 아닌사람들을 나누는것은
개발자가 아닌사람들은 UI를 통해서 시스템을 제어하는것이고
개발자는 UI+API를 이용해서 맥락에 맞게 응용해서 애플리케이션(우리는 웹브라우저)를 제어하는것이다.


웹브라우저를 개발자들도 ui나 모든 동작방법을 처음부터 끝까지 만들지 않는다
>> 웹브라우저를 동작시키는 운영체제개발자들이 버튼, 아이콘등등을 미리 만들어놓고 웹개발자가 사용하게끔(조합하게끔) 미리 만들어준것
운영체제가 제공하는 API를 사용해서 웹브라우저를 만드는 것이다.

계층적인관계의 접점을 Interface라고 한다(층과 층사이에 있는것)

레퍼런스 : 사전과같은것
언어의 사전
자바스크립트의 레퍼런스는 크게 2가지로 나뉜다
1. 자바스크립트자체api
- ex)날짜를 알아내는 api, Math를 통해서 수학적 계산을 하는 api, 정규표현식 2. 자바스크립트가 동작하는 '호스트환경의 api'
- 호스트환경 : 웹브라우저, node.js, appsScript
